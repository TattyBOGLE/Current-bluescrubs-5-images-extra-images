All medical images in the question bank must be clinically accurate and 
educationally valid. A medical student or clinician must be able to look 
at the image and correctly identify the condition from it alone.

─────────────────────────────────────────
1. TWO-STAGE GENERATION PIPELINE
─────────────────────────────────────────
Stage 1 — Claude writes a clinically validated prompt:
Stage 2 — DALL-E 3 HD generates the image from that prompt

NEVER skip Stage 1. Generic prompts produce generic images.
Claude must specify exact pathological features for every condition.

─────────────────────────────────────────
2. CLAUDE SYSTEM PROMPT FOR STAGE 1
─────────────────────────────────────────
Use this exact system prompt when calling Claude to write image prompts:

  "You are a clinical medical educator with expertise in visual diagnosis. 
  Your job is to write precise image generation prompts that produce 
  clinically accurate, educationally valid medical images.

  For each condition you must include ALL of the following in the prompt:
  - Exact anatomical location affected
  - Specific pathological features (colour, texture, morphology, distribution)
  - Classic diagnostic signs that differentiate this condition from mimics
  - Correct skin tone representation where relevant
  - Appropriate clinical context (body part, patient age group if relevant)
  - Imaging modality details if radiology (e.g. PA erect CXR, AP AXR)
  - ECG lead specifics if cardiology

  Output ONLY the image generation prompt. No explanation. No preamble."

─────────────────────────────────────────
3. CONDITION-SPECIFIC CLINICAL REQUIREMENTS
─────────────────────────────────────────
Store a clinicalFeatures object for each question. Pass this to Claude 
alongside the condition name. Examples:

  Malignant Melanoma:
    features: "asymmetric lesion, irregular border, variegated colour 
    (brown/black/red/white), diameter >6mm, on sun-exposed skin"
    must_show: "ABCDE criteria visible"

  Herpes Zoster (Shingles):
    features: "unilateral dermatomal distribution, grouped vesicles on 
    erythematous base, does NOT cross midline, thoracic dermatome"
    must_show: "clear dermatomal boundary, vesicle clusters"

  Finger Clubbing:
    features: "loss of nail bed angle (Schamroth's sign), increased 
    curvature of nail, drumstick appearance of distal phalanx"
    must_show: "Schamroth's window test view preferred"

  Tension Pneumothorax (CXR):
    features: "tracheal deviation AWAY from affected side, absent lung 
    markings on affected side, depressed hemidiaphragm, mediastinal shift"
    must_show: "tracheal deviation is the key sign"

  Complete Heart Block (ECG):
    features: "P waves and QRS complexes completely dissociated, 
    P rate faster than QRS rate, regular PP intervals, regular RR 
    intervals, wide QRS escape rhythm"
    must_show: "clear AV dissociation across all leads"

  Add equivalent clinicalFeatures for every condition in the bank.

─────────────────────────────────────────
5. CLINICAL ACCURACY VALIDATION
─────────────────────────────────────────
After each image is generated, make a second Claude call to validate it:

  Prompt to Claude: 
  "You are a clinician reviewing a medical image for educational accuracy.
  This image was generated for: [CONDITION_NAME]
  Required features: [clinicalFeatures]
  
  Based on the image prompt used, would a medical student be able to 
  correctly identify this condition? 
  List any missing or incorrect clinical features.
  Reply with: PASS or FAIL, then reason."

  IF FAIL → regenerate with Claude's feedback appended to the prompt
  IF PASS → cache and display
  Maximum 3 regeneration attempts before flagging for manual review.
