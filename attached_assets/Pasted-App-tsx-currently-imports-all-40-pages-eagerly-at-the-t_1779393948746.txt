App.tsx currently imports all 40 pages eagerly at the top of the file,
meaning every page's code is bundled and downloaded on first load even
if the user never visits those pages. Add React.lazy() and Suspense to
split the bundle so each page only loads when navigated to.

STEP 1 — CONVERT ALL PAGE IMPORTS TO LAZY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Replace every static page import at the top of App.tsx with a 
React.lazy() dynamic import.

Before:
  import Home from "@/pages/home";
  import Dashboard from "@/pages/dashboard";
  import PLAB1New from "@/pages/plab1-new";
  ... (all 40 pages)

After:
  import { lazy } from "react";

  const Home = lazy(() => import("@/pages/home"));
  const Dashboard = lazy(() => import("@/pages/dashboard"));
  const PLAB1New = lazy(() => import("@/pages/plab1-new"));
  ... (all 40 pages, same pattern)

Keep these imports as static (NOT lazy) — they are needed 
immediately on every render:
  - Navigation component
  - Footer component  
  - QueryClientProvider and queryClient
  - TooltipProvider
  - Toaster
  - I18nProvider
  - NotFound page (used as the catch-all route — must always 
    be available synchronously)

STEP 2 — CREATE A LOADING FALLBACK
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Create a simple PageLoader component inline at the top of App.tsx 
(no need for a separate file):

  function PageLoader() {
    return (
      <div className="min-h-screen flex items-center justify-center 
                      bg-gray-50">

                          border-t-transparent rounded-full

    );
  }

STEP 3 — WRAP ROUTES IN SUSPENSE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

In the Router function, wrap the entire <Switch> in a single 
Suspense boundary:

  import { lazy, Suspense } from "react";

  function Router() {
    return (
      <Suspense fallback={<PageLoader />}>
        <Switch>
          ... all existing routes unchanged ...
        </Switch>
      </Suspense>
    );
  }

Use ONE Suspense boundary around the whole Switch — not one 
per route. This is simpler and avoids layout flicker when 
the fallback swaps in and out.

STEP 4 — PRIORITISE THE MAIN ROUTE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The "/" route renders PLAB1New which is the heaviest page and 
the one users land on first. To avoid a loading flash on the 
very first visit, preload it immediately after the app mounts:

  Add this near the top of the Router component:

  useEffect(() => {
    // Preload the main quiz page in the background 
    // immediately so it's ready before the user navigates
    import("@/pages/plab1-new");
    import("@/pages/dashboard");
  }, []);

This triggers the dynamic import in the background without 
blocking the initial render.

STEP 5 — HANDLE CHUNK LOAD FAILURES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Add an error boundary around the Suspense to catch cases where 
a chunk fails to load (e.g. network error, deploy mid-session):

  Create a simple class component ErrorBoundary in App.tsx:

  class ChunkErrorBoundary extends React.Component
    { children: React.ReactNode },
    { hasError: boolean }
  > {
    constructor(props: any) {
      super(props);
      this.state = { hasError: false };
    }

    static getDerivedStateFromError() {
      return { hasError: true };
    }

    render() {
      if (this.state.hasError) {
        return (
          <div className="min-h-screen flex items-center 
                          justify-center bg-gray-50">
            <div className="text-center">
              <p className="text-gray-600 mb-4">
                Something went wrong loading this page.
              </p>
              <button
                onClick={() => window.location.reload()}
                className="px-4 py-2 bg-blue-600 text-white 
                           rounded-lg text-sm hover:bg-blue-700"
              >
                Reload
              </button>
            </div>
          </div>
        );
      }
      return this.props.children;
    }
  }

  Wrap the Router with it in the App component:

  export default function App() {
    return (
      <ChunkErrorBoundary>
        <QueryClientProvider client={queryClient}>
          <I18nProvider>
            <TooltipProvider>
              <Toaster />
              <Router />
            </TooltipProvider>
          </I18nProvider>
        </QueryClientProvider>
      </ChunkErrorBoundary>
    );
  }

Check vite.config.ts. If a build.rollupOptions.output 
section exists, ensure it does not force everything into 
a single chunk. The default Vite behaviour with React.lazy() 
will automatically create separate chunks — do not override 
this with manualChunks unless it already exists, in which 
case add page chunks:

  manualChunks: {
    'vendor-react': ['react', 'react-dom'],
    'vendor-ui': ['@radix-ui/react-dialog', 
                  '@radix-ui/react-select',
                  'lucide-react'],
    'vendor-query': ['@tanstack/react-query'],
  }

Only add manualChunks if the config does not already have it.
